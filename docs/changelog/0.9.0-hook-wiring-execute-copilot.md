# 0.9.0 — Hook Wiring, Execute Flag, Copilot Research

**Date:** 2026-07-24

## Added

- **`scaffold.py wire_hooks()`** — reads `hooks-manifest.json`, rewrites hook paths from framework layout to `.ai-badger/skills/`, merges registrations into `.claude/settings.json`. Idempotent — skips hooks already registered.
- **`scaffold.py --execute` flag** — runs skill install commands via `subprocess.run` with 30s timeout per command and error logging. Default behavior unchanged (advisory/prints only).
- **`prompt-markers` hook entry** in `hooks-manifest.json` for `UserPromptSubmit` event.
- **`features/common/support.json`** — agent capability matrix mapping what each agent supports and what ai-badger scaffolds. Covers Claude Code, Hermes, Copilot CLI, and Junie.
- **Copilot support plan** at `.hermes/plans/copilot-support-plan.md` — 3-phase plan for hooks, skills, and custom agent support.

## Fixed

- **known-gaps.md**: marked gaps 1 (hooks auto-wire) and 6 (install advisory) as resolved.
- 5 new tests: hook wiring (3) + execute flag (2).

## Research

- Copilot CLI now supports hooks (`.github/hooks/*.json` with 8 events), skills (`.github/skills/*/SKILL.md`), and custom agents (`.github/agents/*.agent.md`). The article's feature matrix is outdated — Copilot has `—` for hooks but should show `native`.
