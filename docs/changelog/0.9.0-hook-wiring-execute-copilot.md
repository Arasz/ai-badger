# 0.9.0 — Adjustment Execution, Copilot Support via Adjustments

**Date:** 2026-07-24

## Added

- **`scaffold.py run_adjustments()`** — executes agent-specific adjustment scripts during scaffold. Each adjustment is a Python script with an `adjust(context)` function that receives framework root, config, and target directory. Adjustments are declared in `features/<agent>/adjustments/adjustment.json`.
- **Copilot hooks adjustment** (`features/copilot/adjustments/adjust_hooks.py`) — wires ai-badger hooks into `.github/hooks/ai-badger-hooks.json` in Copilot CLI format (`{ "version": 1, "hooks": {...} }`). Rewrites paths from framework layout to `.ai-badger/skills/`.
- **Copilot skills adjustment** (`features/copilot/adjustments/adjust_skills.py`) — symlinks ai-badger skills into `.github/skills/` for Copilot CLI discovery. Skills must have `SKILL.md`.
- **Copilot agents adjustment** (`features/copilot/adjustments/adjust_agents.py`) — maps ai-badger personas to Copilot custom agents (`.github/agents/*.agent.md`) with YAML frontmatter (uses `yaml.dump` for safe serialization).
- **`hooks-manifest.json`**: added `copilot` entries for `drift-notice` (sessionStart) and `prompt-markers` (userPromptSubmitted).
- **`features/common/support.json`** — agent capability matrix. Copilot hooks, skills, and custom agents now show `aiBadgerSupport: true`.

## Changed

- **`scaffold.py`**: `run()` now calls `run_adjustments()` after `wire_hooks()` and before `install_plugins()`.
- **`scaffold.py wire_hooks()`** — reads hooks-manifest.json, rewrites paths, merges into `.claude/settings.json` (idempotent).
- **`scaffold.py --execute` flag** — runs install commands via subprocess with 30s timeout and error logging.

## Fixed

- **hooks-manifest.json**: Copilot entries were missing — `adjust_hooks.py` was always returning "No Copilot hooks to wire".
- **adjust_hooks.py**: Event name mapping fixed — manifest uses camelCase (Copilot format), source hooks.json uses PascalCase (Claude format).
- **adjust_agents.py**: YAML frontmatter now uses `yaml.dump()` instead of f-string (avoids breakage on special characters).
- 5 new tests: hook wiring (3) + execute flag (2).
