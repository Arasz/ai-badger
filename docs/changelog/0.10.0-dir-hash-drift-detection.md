# 0.10.0 — Directory-Content Hash Drift Detection

**Date:** 2026-07-24

## Changed

- **Drift detection for skills (directory entries)**: Replaced commit-based approach (v0.9.3) with two-phase directory-content hash comparison:
  1. **Structural pre-check** (O(1)): compare file count + subdir count from `dirMeta`
  2. **Content hash**: SHA-256 of sorted (relative_path + file_content), excluding tests/evals/__pycache__
- **`badger_lib.dir_content_hash()`**: New function computing structural fingerprint + content hash for directories, with configurable exclude patterns.
- **`badger_lib.SKILL_EXCLUDE_PATTERNS`**: Shared constant matching `scaffold.py`'s `_test_ignore`.
- **`scaffold.py record()`**: Now hashes SOURCE dir (before extension embedding) for directory entries, storing `dirMeta` (file_count, dir_count) in manifest.
- **`drift.py compare()`**: Uses `dir_content_hash()` with structural pre-check instead of commit comparison.
- **`detect_additions.py`**: Updated to use `dir_content_hash()` for directory entries, excluding extensions/ (config-driven, not project additions).
- **`refresh.py`**: Removed `framework_commit` parameter and git subprocess call — no longer needed.

## Removed

- **Commit-based drift detection** (v0.9.3): `framework_commit` parameter removed from `compare()` and `run_drift()`. Directory hash comparison is more precise — only flags skills whose actual content changed.

## Manifest format

- Directory entries now include `dirMeta: {file_count, dir_count}` alongside `hash`.
- Backward compatible: manifests without `dirMeta` fall back to content hash comparison.

## Tests

- 4 new tests for `dir_content_hash()`: excludes patterns, deterministic, content change detection, excluded files don't affect hash.
- 2 new tests for `compare()`: detects changed dir by hash, passes unchanged dir.
- 2 existing tests updated to reflect new directory hash behavior.
- 511 tests pass (up from 504).
