# 0.9.4 — fix adjust_task.py absolute path in record()

## What changed

- **Fix:** `features/hermes/adjustments/adjust_task.py` returned `files: [str(ext_md)]` with an
  absolute path to `extension.md`. When `scaffold.py`'s `run_adjustments()` passed this to
  `record()`, `Path.relative_to()` raised `ValueError` because the absolute path was not under
  the target directory. Changed to `files: []` — the adjustment doesn't copy files; the actual
  extension embedding is handled by `_embed_extensions()`.

- **Lint:** Fixed all pylint warnings across `scripts/`, `features/`, and `tests/`:
  - `scripts/badger_lib.py`: added explicit `check=False` to `subprocess.run` calls
  - `features/common/skills/welcome-ai-badger/scripts/scaffold.py`: added `check=False`,
    removed duplicate `import shutil`, added `too-many-statements` to disabled complexity
    thresholds
  - `features/common/skills/den-refresh/scripts/refresh.py`: added `check=False`
  - Updated lint command from `pylint scripts skills tests` to `pylint scripts features tests`
    (the `skills/` directory no longer exists at the root)
