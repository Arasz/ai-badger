# 0.9.2 — den-refresh Version Sync Without Drift

**Date:** 2026-07-24

## Fixed

- **`refresh.py` version sync**: `config.frameworkVersion` is now updated whenever `scaffold_version != current_version`, regardless of whether file drift was detected. Previously, the version sync was gated inside the drift/re-scaffold block, so a framework version bump with no file changes (internal refactors, test-only releases) left the project permanently on the old version string.
- **TDD**: failing test written first (`test_refresh_updates_framework_version_when_no_drift_but_version_bumped`), then fix applied.
