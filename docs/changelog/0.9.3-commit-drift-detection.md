# 0.9.3 — Commit-Based Drift Detection for Skills

**Date:** 2026-07-24

## Fixed

- **`drift.py compare()`**: directory-valued entries (skills) are now reported as "changed" when the framework's git commit differs from the manifest's `frameworkCommit`. Previously, directory entries were always "skipped" because hash comparison doesn't work on directories that strip tests/evals during scaffold. This meant skills with updated SKILL.md or scripts/ were never refreshed unless there was also file-level drift elsewhere.
- **`refresh.py`**: passes the framework's current HEAD commit to `drift.compare()` for commit-based directory drift detection.
- **Evals clarification**: `evals/` directories are correctly stripped during scaffold by `_test_ignore` — they are framework-only quality-regression content and are never distributed to scaffolded projects.
