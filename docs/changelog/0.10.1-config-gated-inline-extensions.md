# 0.10.1 — config-gated inline extensions

## What changed

- Task skill extensions (`github`, `hermes`) moved from external `features/<stack>/skills/task-extensions/`
  to inline `features/common/skills/task/extensions/` with `extension.json` activation conditions.
- `scaffold.py`: added `_prune_inline_extensions()` to gate inline extensions by config requirements
  after `copytree`. Old external `task-extensions/` directories removed.
- `requirement_met()` now handles both `==` and `=` value-equality syntax in extension requirements.
- Manifest directory hash now computed from the TARGET (scaffolded output) instead of SOURCE,
  so `detect_additions` sees the actual scaffolded state.
- `index.json` regenerated: `task` skill no longer lists `extensions` key (now inline).
- `adjust_task.py` path updated to new inline location.

## Why

The PR added inline extension files but the old external directories were still being found by
`_embed_extensions`'s glob, bypassing the `requires` gate. The hermes external extension had no
`extension.json` so it embedded unconditionally regardless of config.
