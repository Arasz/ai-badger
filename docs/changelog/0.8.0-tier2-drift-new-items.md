# 0.8.0 — Tier 2 Drift: New Items Detection

**Date:** 2026-07-24

## Added

- **`drift.detect_new_items()`** — walks `index.json` for the project's configured stacks and finds catalog items not present in the manifest. Reports as `newItems` in drift output.
- **`drift.compare()` new `stacks` parameter** — when provided, includes `newItems` in the result by calling `detect_new_items()`.
- **den-refresh** now passes config stacks to drift detection and considers new items as drift (triggers re-scaffold).
- **drift.py CLI** reads config stacks automatically and reports new items in output.

## Fixed

- Addresses issue #25: ADR-0001 decision 5 "new items" detection is now implemented.
- Non-breaking upstream version awareness: drift.py prints scaffolded vs current version on every run.
